# docs(examples): standardize roxygen example formatting and update conventions

**Date**: 2026-04-29
**Branch**: docs/roxygen-examples-cleanup
**Phase**: Phase 0.7

## Changes

- Reformatted every `@examples` block across 20 R/ files to follow the new
  style rule: 80-char width, 2-space indent, long calls broken one-arg-per-line,
  pipelines expanded across lines, each logical block prefaced with a short
  `# ...` comment, blank `#'` lines between blocks.
- Dropped `library(dplyr)` from all examples. The dplyr verbs surveytidy uses
  (mutate/select/filter/arrange/group_by/joins/etc.) and `tidyr::drop_na` are
  re-exported via `R/reexports.R`, so they work bare under just
  `library(surveytidy)`. Loading dplyr was actively harmful: it masked
  surveytidy's own `case_when()`, `if_else()`, `na_if()`, `recode()`, and
  `bind_rows()`. The case_when example was failing R CMD check for exactly
  this reason.
- Namespace-prefixed helpers that surveytidy does NOT re-export:
  `dplyr::if_any/if_all/desc/across/c_across/cur_column` and
  `tidyselect::starts_with/ends_with/contains/where/all_of/any_of`.
- Replaced `\dontrun{}` blocks for `right_join`/`full_join`/`bind_rows` with
  runnable demonstrations: each error now caught with `tryCatch()` so the
  message prints inline, plus a working alternative (`left_join` after the
  join errors; `survey_data() → bind_rows() → as_survey()` workflow after
  the bind_rows error). Both verified to execute cleanly.
- Removed phase-related stale content from CLAUDE.md (Phase 0.5 status line,
  Phase 1 estimation references, Phase 0.5 Build Order section).
- Refreshed CLAUDE.md against the actual v0.5.0 state: broadened Purpose to
  list recoding/rowwise/joins/collection-dispatch; replaced stale R/ and
  tests/ trees with the current contents grouped by family; expanded the
  Skills section from 3 entries to all 14 (planning/implementation/review/
  release groups); fixed the rename() docs to reference the `.sc_*` wrappers
  in `R/utils.R` instead of the banned `surveycore:::*` calls; added an
  explicit `:::` policy block; corrected the github-strategy.md path; softened
  the survey-standards reference; added a Common Commands snippet.
- Flipped the helper-placement rule in `code-style.md`: single-use helpers now
  live BELOW the exported functions in the same file (R parses the file fully
  before any call, so order doesn't matter for resolution). Replaced three
  stale `07-utils.R` references with `R/utils.R`.
- Added a new `@examples`: style and formatting subsection to
  `r-package-conventions.md`, including the bad/good template and the explicit
  ban on `library(dplyr)` plus the namespace-prefix list.
