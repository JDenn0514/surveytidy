# Extract a column from a survey design object

Pull a single column out of a survey design object as a plain vector.
This is a terminal operation — the result is not a survey object and
cannot be piped back into survey verbs.

## Usage

``` r
pull.survey_base(.data, var = -1, name = NULL, ...)
```

## Arguments

- .data:

  A
  [`survey_base`](https://jdenn0514.github.io/surveycore/reference/survey_base.html)
  object.

- var:

  \<[`data-masking`](https://rlang.r-lib.org/reference/args_data_masking.html)\>
  The column to extract. Accepts a bare name, a positive integer
  (counting from the left), or a negative integer (counting from the
  right). Defaults to the last column.

- name:

  \<[`data-masking`](https://rlang.r-lib.org/reference/args_data_masking.html)\>
  An optional column whose values are used as names for the returned
  vector.

- ...:

  Passed to
  [`dplyr::pull()`](https://dplyr.tidyverse.org/reference/pull.html).

## Value

A vector the same length as the number of rows in `.data`.

## See also

[`select()`](https://dplyr.tidyverse.org/reference/select.html) to keep
columns in the survey object

Other selecting:
[`glimpse.survey_base()`](https://jdenn0514.github.io/surveytidy/reference/glimpse.survey_base.md),
[`relocate.survey_base()`](https://jdenn0514.github.io/surveytidy/reference/relocate.survey_base.md),
[`select.survey_base()`](https://jdenn0514.github.io/surveytidy/reference/select.survey_base.md)
